# Security policy

## Supported versions

Security fixes are applied to the latest published version. Users should
upgrade to the newest release before reporting a defect.

## Reporting a vulnerability

Use
[GitHub private vulnerability reporting](https://github.com/Pisama-AI/n8n-nodes-pisama/security/advisories/new).
Do not open a public issue for vulnerabilities, credentials, workflow data, or
customer information.

Include the affected package version, n8n version, impact, and a minimal
reproducer with all secrets removed. Pisama will acknowledge a valid report
within five business days.
